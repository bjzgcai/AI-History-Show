# 成就名称

逻辑理论家 / 逻辑理论机器（Logic Theorist / Logic Theory Machine）

# 年份 / 时期

1956

# 类型

推理

# 一句话总结

Logic Theorist 是一个早期符号 AI 程序，它用启发式问题求解方法搜索数理逻辑定理的证明。

- description: 1956 年，艾伦·纽厄尔、赫伯特·A·西蒙和 J. C. Shaw 构建了 Logic Theorist，用来发现符号逻辑中的证明。它把定理证明视为由启发式规则引导的搜索问题，而不是穷举计算。这个系统成为自动推理、符号 AI 和信息加工认知观的重要里程碑。

# 历史背景

在 1956 年达特茅斯会议正式命名“人工智能”之前，研究者已经在讨论计算机能否操纵符号、解决问题并模拟人类思维。纽厄尔和西蒙关注人类问题求解、决策过程，以及计算机是否能模拟认知过程，而不仅仅做数值计算。

Logic Theorist 就产生于这个背景。它处理 Russell 和 Whitehead 的《Principia Mathematica》中的定理，尝试用符号变换构造证明链。它的重要性不只是得到答案，而是把“智能问题求解过程”本身变成可计算、可观察的对象。

# 核心思想

Logic Theorist 把定理证明看作搜索。给定一个要证明的定理，它尝试使用推理规则和符号替换，从已知公理逐步接近目标命题。

它的关键创新是启发式引导。程序不是盲目尝试所有可能证明路径，而是用经验规则优先选择更有希望的变换，使自动推理更接近人类问题求解。

# 关键概念

- 启发式搜索: Logic Theorist 用经验规则引导证明搜索。这让推理成为选择性过程，而不是穷举所有可能证明。
- 符号推理: 程序把逻辑表达式作为符号来操作。这成为早期 AI 系统的核心模式之一。
- 自动定理证明: 系统尝试为数理逻辑命题发现证明链。它帮助确立了定理证明作为 AI 重要任务的地位。

# 影响力

## 学术影响

BenchCouncil 将《The Logic Theory Machine: A complex information processing system》列为 1,046 次引用。该工作影响了自动定理证明、符号 AI、认知模拟、启发式搜索和后来的 General Problem Solver。ACM 图灵奖页面也将 Newell 和 Simon 的贡献归入人工智能、人类认知心理学和表处理等基础方向。

## 产业影响

Logic Theorist 本身不是商业产品，但它影响了符号 AI 系统的工程方法：把知识表示为符号结构，在可能变换之间搜索，并用启发式规则降低组合爆炸。这些模式后来进入定理证明器、规划系统、专家系统和 AI 编程语言。

## 长期遗产

Logic Theorist 被视为最早的 AI 程序之一，也是机器能够进行符号推理的早期展示。它留下的不是单一产品，而是一种设计模式：智能可以被理解为在符号表示空间中由启发式规则引导的搜索。
