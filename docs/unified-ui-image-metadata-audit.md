## 本次发现的问题

### 内部状态被当作图片类型

52 个迁移资产使用了 `external-reference-image` 或 `external-reference-diagram` 作为角色。编译器又会在缺少副标题时把角色写入可见说明，导致内部状态直接显示在图片下方。

这类字段现已改为真实的语义类型，例如人物肖像、历史资料图、系统照片、程序截图、算法解释图和架构解释图。图片是否来自外部地址仍保留在版权与来源信息中，不再作为面向观众的图片类型。

### 通用标题与错误类型

- 19 张图片被统一标成“人物肖像 / 相关研究者照片”，实际内容包含人物与模型组合图、视频截图、人物资料卡、团队照片和演讲照片。
- 9 张图片没有明确副标题，其中包括 AlexNet 的网页截图、错误率演进图、Transformer 自注意力图和论文首页。
- 43 张图片使用“结构示意”“历史照片”“论文页面”或“项目使用的架构图”等通用文案，无法说明图片实际内容。

上述图片均已改成具体对象名称和用途说明。同一事件存在多张同类图片时，标题会使用页码或序号区分。

## 不再用于统一 UI 的图片

以下资产没有删除，只从事件的展示选择中移除：

- 反向传播事件中重复且带播放器界面的 Geoffrey Hinton 视频截图。
- BERT 事件中与原始论文作者无直接关系的 Andrej Karpathy 人物资料卡。
- 第一代 GPT 事件中时间较晚的 Sam Altman 访谈画面。

KNN 与 HOG 已改用从 Stanford ISL 和 INRIA/LEAR 作者主页取得的本地人物图。GNN 已用锡耶纳大学作者主页的弗兰科·斯卡塞利肖像替换 Google Scholar 默认头像，Conditional GAN 已用论文首页替换身份未核实的人物图。其余当前展示图片也已统一下载到 `resources/images/external/<event-id>/`。

## 校验规则

Archive 校验现在要求所有图片、SVG 和 GIF：

- 必须具有中英文标题和副标题。
- 必须使用语义图片角色，不能使用 `external-reference-image` 或 `external-reference-diagram` 作为图片类型。
- 可见字段不能包含内部角色代码。
- 可见字段不能使用“外部参考图”“人物肖像”“相关研究者照片”“结构示意”“历史照片”“论文页面”等无具体对象的占位文案。
- 可见说明不能使用“项目使用的架构图”这类项目内部描述。

统一 UI 读取元数据时，Archive `imageMeta` 会覆盖旧兼容字段 `resources.imageMeta`。前端也不再使用图片内部角色作为副标题兜底。
