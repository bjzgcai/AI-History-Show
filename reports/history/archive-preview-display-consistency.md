# Archive Preview Display Consistency Analysis

This report explains why archive preview data previously differed from the normal main exhibit, what was fixed, and what differences intentionally remain.

## Current status

After the latest cleanup, forced archive presentation preview no longer changes the main visible presentation fields.

```text
Normal milestones: 134
Archive preview milestones: 134
Changed milestones: 134
Changed fields: sources: 134, commentary: 131, quizzes: 112
Image decreases: 0
Source decreases: 0
Commentary decreases: 0
Quiz decreases: 0
```

The remaining differences are structural archive data replacements only. The following display-critical fields now match the normal generated exhibit:

```text
title: 0 differences
subtitle: 0 differences
description: 0 differences
visual: 0 differences
images: 0 differences
analysis: 0 differences
```

## Why archive-review looked consistent while the main preview differed

`archive-review.html` and `?archivePreview=1` answer different questions.

### Normal generation / archive-review

Normal generation uses `presentationMode: preserve-legacy` by default.

```text
legacy presentation + archive metadata = generated final
```

This is what `archive-review.html` reviews. It is designed to prove that archive overlay does not unexpectedly change the current public exhibit.

### Forced archive preview mode

`?archivePreview=1` loads `milestones-data-archive-preview.js`, produced by:

```bash
npm run preview:archive-data
```

That file is generated by copying the normal generated data and then forcing archive presentation overlay for all archive-covered milestones. It is a local preview of what would happen if archive data directly drove the main exhibit.

Therefore, early mismatches in `?archivePreview=1` were expected diagnostic output, not proof that normal exhibit generation was broken.

## Root causes of the previous display mismatches

### 1. AI100-only events used external legacy image URLs not represented as archive assets

Many AI100 legacy milestones mixed local `resources/...` images with remote URLs, for example Wikimedia, Google Scholar, university pages, company pages, and personal pages.

When archive preview initially used only archive assets, it kept local governed images but dropped remote legacy images.

Observed before cleanup:

```text
AI100-only image decreases: 42 events
Missing images: 47
Remote missing images: 47
Local missing images: 0
```

This showed the issue was not a local path lookup bug. The missing images were external legacy references that had not been represented in archive assets.

Fix:

- Added those 47 remote images to their event `assets.json` files.
- Marked each as `rights.status: external-reference`.
- Did not download or copy them into the repository.
- Added the corresponding asset IDs to AI100 variants.
- Updated archive validation to accept `http://` and `https://` asset paths as external references.

### 2. Shared multi-branch events had the same external URL gap

After AI100-only image parity was restored, a smaller set of shared events still had image decreases:

```text
1989-cnn
2020-alphafold
1951-strachey-draughts
1994-chinook
```

These were not AI100-only events; they also appeared in deep-learning or gaming variants. The underlying cause was the same: external legacy images were not represented in archive assets.

Fix:

- Added shared external-reference assets for the missing remote images.
- Added them to every affected variant, e.g. both AI100 and gaming variants when applicable.

### 3. Curated variants had incomplete `assetIds`

Some curated events already had the needed local images in `assets.json`, but their specific variants did not select all legacy images.

Affected examples:

```text
2012-alexnet / deep-learning
2017-transformer / deep-learning
2017-transformer / bench-council-ai100
```

Fix:

- Added missing existing asset IDs to the relevant variants.
- Added `asset-transformer-architecture-detail` for `resources/images/2017-transformer/architecture/2017-transformer_architecture_02.png`.
- Restored image order to match legacy presentation.

### 4. Curated variants had fewer source IDs than legacy presentation

Some deep-learning variants used a curated subset of sources while legacy presentation showed more.

Affected examples:

```text
2012-alexnet / deep-learning
2017-transformer / deep-learning
```

Fix:

- Restored the missing source IDs in the archive variants.
- Current source decreases are now zero.

### 5. AlphaGo AI100 commentary had only two archive sections

`2016-alphago / bench-council-ai100` archive commentary had:

```text
published-paper
long-term-legacy
```

Legacy and AI100 content expectations require the three-part structure:

```text
Historical Background
Core Idea
Long-Term Legacy
```

Fix:

- Reworked the AlphaGo AI100 variant commentary into the expected three sections.
- Merged the previous paper-focused content into `Core Idea`.
- Current commentary decreases are now zero.

### 6. Archive variants intentionally rewrote presentation fields

Some variants had archive-specific titles, subtitles, descriptions, visuals, or analysis intended to demonstrate storyline-specific narration.

Examples before restoration:

```text
2012-alexnet / deep-learning
2017-transformer / deep-learning
2016-alphago / gaming-ai
```

These were not data loss, but they did make the forced archive preview visually differ from normal exhibit output.

Decision:

- Do not keep archive-specific presentation changes for now.
- Restore archive variants to use the original legacy presentation fields.

Fix:

- Restored legacy `displayTitle`, `displaySummary`, `displayDescription`, `visual`, `analysis`, and image ordering for all affected variants.

## What differences remain

The current forced archive preview still differs from normal data in these fields:

```text
sources
commentary
quizzes
```

These are structural data differences:

- Archive source objects include archive IDs and provenance-oriented metadata.
- Archive commentary objects are generated from variant files.
- Archive quiz objects are archive-normalized.

They do not currently reduce counts:

```text
Source decreases: 0
Commentary decreases: 0
Quiz decreases: 0
```

They also no longer change display-critical fields:

```text
title/subtitle/description/images/visual/analysis: no differences
```

```text

```

Normal archive migration safety is still checked by:

```bash

npm run build:archive
npm run generate
npm run review:archive
npm run diff:archive
```

## Policy going forward

1. Normal generation must remain `preserve-legacy` unless a display change is explicitly approved.
2. Remote legacy images may be represented as `external-reference` archive assets for parity, but should not be downloaded into the repository without license review.
3. Archive preview should not reduce image/source/commentary/quiz counts relative to legacy.
4. Any future archive-specific title, description, visual, or analysis change should be reviewed as an intentional presentation change, not introduced through batch migration.
